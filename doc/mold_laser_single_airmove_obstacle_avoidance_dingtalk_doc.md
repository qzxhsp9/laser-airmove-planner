# 模具激光加工中单段空移避障技术调研

## 1. 背景说明

在模具激光加工过程中，激光头在非工作状态下从一个位置移动到另一个位置，这类运动通常称为：

- 空程；
- 空移；
- 非加工移动；
- rapid traverse；
- air move。

本阶段研究对象限定为：

> 单段空移避障，即激光头从一个起点安全、高效地移动到一个终点，不考虑全局切割顺序优化。

该问题主要用于以下模具激光加工场景：

- 模具激光修复；
- 模具激光熔覆；
- 模具激光淬火；
- 模具激光纹理加工；
- 模具激光清洗；
- 深腔模具内部激光加工；
- 五轴激光加工设备中的非加工移动。

在这些场景中，空移路径的质量会直接影响：

- 加工效率；
- 碰撞风险；
- 设备稳定性；
- 运动平顺性；
- 导轨、丝杆、电机等机械寿命；
- 激光头、喷嘴、保护镜等关键部件安全。

---

## 2. 问题定义

### 2.1 输入条件

单段空移避障通常需要以下输入：

- 起点位置或位姿；
- 终点位置或位姿；
- 模具三维模型，例如 STL / STEP / mesh；
- 夹具、工装、工作台等障碍物模型；
- 激光头几何包络；
- 喷嘴长度与半径；
- TCP 与实际碰撞体之间的偏置关系；
- 安全距离；
- 运动空间边界；
- 速度、加速度、jerk 限制。

### 2.2 输出结果

系统需要输出一条满足以下条件的轨迹：

- 从起点到终点；
- 不与模具、夹具、工装、工作台发生碰撞；
- 满足最小安全距离；
- 尽可能短；
- 尽可能平滑；
- 满足设备运动学与动力学约束；
- 可被 CNC、机器人控制器或上位机执行。

### 2.3 核心目标

该问题不是单纯求一条几何路径，而是综合考虑：

```text
安全性 + 效率 + 平滑性 + 可执行性
```

因此，完整系统通常需要同时解决：

1. 几何避障；
2. 路径优化；
3. 曲线平滑；
4. 速度规划；
5. jerk 限制；
6. 轨迹可执行性验证。

---

## 3. 行业主流技术路线

目前行业中较完整的技术路线一般可以抽象为：

```text
三维建模
  ↓
碰撞检测
  ↓
单段路径规划
  ↓
路径平滑
  ↓
速度/加速度/jerk 规划
  ↓
轨迹输出与执行
```

对应到当前推荐技术栈：

```text
几何避障：OMPL + FCL
运动执行：Ruckig
轨迹表达：Bezier / B-spline
```

## 4. 主要技术一：几何建模与碰撞检测

### 4.1 技术内容

模具激光加工中的避障首先需要建立几何环境，包括：

- 模具表面；
- 夹具；
- 工装；
- 工作台；
- 禁行区域；
- 激光头几何包络；
- 喷嘴几何包络；
- 五轴摆头外形。

常见几何表达包括：

- STL 三角网格；
- STEP / B-Rep 模型；
- voxel 栅格；
- signed distance field，简称 SDF；
- bounding box；
- cylinder；
- capsule；
- convex hull。

### 4.2 常用碰撞检测方法

常用方法包括：

1. 三角网格碰撞检测；
2. 包围盒层次结构 BVH；
3. 距离查询；
4. 安全距离检测；
5. 碰撞体膨胀；
6. 分层粗筛 + 精检测。

工程中通常不会直接用复杂激光头全模型做实时碰撞检测，而是使用简化几何包络，例如：

- 圆柱近似喷嘴；
- 胶囊体近似光路或喷嘴包络；
- box 近似激光头主体；
- 多个 primitive 组合表达完整工具头。

### 4.3 典型开源库

该层常用开源库包括：

- FCL：Flexible Collision Library；
- Bullet；
- Trimesh；
- Open3D；
- CGAL；
- libigl。

其中，FCL 在机器人和运动规划领域使用较广，适合与 OMPL、MoveIt、Tesseract 等框架集成。

## 5. 主要技术二：A* / Dijkstra 搜索

### 5.1 基本思想

A* 和 Dijkstra 属于图搜索算法。应用到空移避障时，通常需要先将三维工作空间离散化为：

- 2D grid；
- 3D voxel；
- 八叉树；
- 多层高度图。

然后在可通行空间中搜索从起点到终点的路径。

### 5.2 优点

- 实现简单；
- 结果稳定；
- 可控性强；
- 容易加入安全距离；
- 适合三轴场景；
- 便于工业调试。

### 5.3 缺点

- 空间离散化精度影响结果；
- 分辨率越高，计算量越大；
- 路径通常是折线；
- 拐点多，轨迹不平滑；
- 高维姿态规划中容易出现维度爆炸。

### 5.4 适用场景

A* / Dijkstra 更适合：

- 三轴激光空移；
- 简单障碍物；
- 已知固定场景；
- 对稳定性要求高、对最优性要求一般的工业系统。

---

## 6. 主要技术三：RRT / RRT* / Informed RRT*

### 6.1 基本思想

RRT 系列算法属于采样式路径规划方法。其核心思想是在连续空间中随机采样，并逐渐扩展搜索树，直到连接起点和终点。

常见算法包括：

- RRT；
- RRTConnect；
- RRT*；
- Informed RRT*；
- BIT*；
- PRM。

### 6.2 优点

- 适合连续空间；
- 适合复杂障碍物；
- 适合高维规划；
- 可处理位姿约束；
- 更适合五轴激光头、机器人激光加工、深腔模具等场景。

### 6.3 缺点

- 路径随机性较强；
- 原始路径通常不平滑；
- 需要后处理；
- 参数调节影响较大；
- 不一定直接满足运动学和动力学约束。

### 6.4 适用场景

RRT 系列适合：

- 深腔模具；
- 五轴激光加工；
- 激光熔覆修复；
- 复杂夹具环境；
- 需要姿态避障的场景。

### 6.5 推荐工程组合

工程中较推荐的组合是：

```text
先尝试直线空移
  ↓
直线失败后使用 RRTConnect 快速找可行解
  ↓
再用 RRT* / Informed RRT* 优化路径
  ↓
最后进行平滑和速度规划
```

## 7. 主要技术四：人工势场法 APF

### 7.1 基本思想

人工势场法将目标点看作引力源，将障碍物看作斥力源。激光头在虚拟力场作用下从起点移动到终点。

### 7.2 优点

- 计算速度快；
- 适合局部实时避障；
- 实现简单；
- 可以与传感器反馈结合。

### 7.3 缺点

- 容易陷入局部最优；
- 狭窄通道表现不稳定；
- 参数敏感；
- 很难单独保证全局可达。

### 7.4 工程定位

人工势场法不建议单独作为主规划器，更适合作为：

```text
全局路径规划后的局部修正方法
```

常见组合为：

```text
A* / RRT 全局规划
  +
APF 局部避障修正
```

---

## 8. 主要技术五：Bezier / B-spline / NURBS 路径平滑

### 8.1 为什么需要路径平滑

A*、Dijkstra、RRT 等算法输出的路径通常是折线路径，例如：

```text
P0 → P1 → P2 → P3
```

这种路径直接执行会带来：

- 拐点处速度突变；
- 加速度突变；
- jerk 冲击；
- 机械振动；
- 跟踪误差增大；
- 高速空移时必须频繁降速。

因此必须对路径进行平滑处理。

### 8.2 常用曲线

常用平滑曲线包括：

- Bezier 曲线；
- B-spline；
- NURBS；
- Catmull-Rom spline；
- quintic polynomial；
- PH curve。

### 8.3 工程要求

路径平滑必须满足：

- 起点和终点不变；
- 平滑后仍不碰撞；
- 与原始路径偏差受限；
- 曲率不能过大；
- 采样点满足最小安全距离；
- 可以进行后续速度规划。

### 8.4 常见工程流程

```text
原始路径
  ↓
删除冗余点
  ↓
shortcut 简化
  ↓
Bezier / B-spline 拟合
  ↓
密集采样
  ↓
碰撞回检
  ↓
不安全区域局部回退
```

## 9. 主要技术六：jerk-limited 轨迹规划

### 9.1 jerk 的含义

jerk 是加速度对时间的变化率，也叫加加速度。

在高速激光设备中，jerk 过大会导致：

- 机械冲击；
- 导轨振动；
- 丝杆或电机负载突变；
- 激光头抖动；
- 轨迹跟踪误差；
- 拐角处加工质量下降。

### 9.2 为什么空移也需要 jerk 控制

虽然空移时激光关闭，但激光头依然在高速运动。若轨迹不平滑或加速度突变，仍会造成设备冲击。

因此，高速空移通常需要：

- S 曲线加减速；
- 加速度连续；
- jerk 限制；
- look-ahead 前瞻速度规划。

### 9.3 常用工具

该层可使用：

- Ruckig；
- TOPP-RA；
- 自研 S 曲线规划器；
- 控制器内部 look-ahead 模块。

### 9.4 推荐工程路线

对于当前项目，可以采用：

```text
OMPL 输出几何路径
  ↓
Bezier / B-spline 平滑路径
  ↓
Ruckig 生成 jerk-limited 轨迹
```

## 10. 行业当前主要痛点

## 10.1 痛点一：能避障，但不够快

很多算法可以找到无碰撞路径，但路径过于保守，导致空移距离长、时间长。

典型问题包括：

- 过度抬高 Z；
- 路径绕行过远；
- 没有利用局部安全通道；
- 没有综合考虑时间最优。

对于深腔模具，该问题更加明显。

---

## 10.2 痛点二：路径可行，但轨迹不可执行

很多规划算法只关注几何无碰撞，但没有考虑设备的实际运动能力，例如：

- 最大速度；
- 最大加速度；
- 最大 jerk；
- 电机响应；
- 控制器插补能力；
- 五轴联动极限。

结果是：

```text
仿真中路径可行，实际设备无法稳定执行。
```

这是目前工业落地中最典型的问题之一。

---

## 10.3 痛点三：几何规划与轨迹规划脱节

传统流程通常是：

```text
先找路径
  ↓
再做平滑
  ↓
再做速度规划
```

这种分阶段方法容易出现问题：

- 路径规划阶段不考虑曲率；
- 平滑后可能碰撞；
- 速度规划后发现某些段无法高速通过；
- 最终轨迹与原始路径差异较大。

未来趋势是：

```text
路径规划 + 平滑 + 动力学约束联合优化
```

---

## 10.4 痛点四：五轴姿态空间复杂

对于五轴激光加工，状态空间不再只是：

```text
X, Y, Z
```

而是：

```text
X, Y, Z, A, B
```

或完整的：

```text
位置 + 姿态 + 机构关节约束
```

这会导致：

- 状态维度升高；
- 搜索空间变大；
- 碰撞检测复杂度提升；
- 姿态插值困难；
- 机械限位和奇异位形问题增加。

因此，五轴场景更适合使用 RRT*、TrajOpt、CHOMP、STOMP 等方法。

---

## 10.5 痛点五：实时性不足

模具加工现场可能存在：

- 工件装夹误差；
- 热变形；
- 熔覆高度变化；
- 表面形貌变化；
- 夹具位置偏差；
- 飞溅物和临时障碍。

这些因素可能需要在线重规划。但复杂三维模型上的实时碰撞检测和路径规划计算量较大，目前仍是难点。

---

## 10.6 痛点六：安全距离难以精确保证

只做碰撞检测并不等于安全。工业中通常还需要保证：

```text
激光头与模具/夹具之间的最小距离 > 安全阈值
```

但安全距离受到以下因素影响：

- 激光头模型简化误差；
- STL 精度；
- 坐标标定误差；
- 设备跟踪误差；
- 模具实际装夹误差；
- 运动过程中的振动。

因此，工程上通常需要设置保守安全裕量。

---

## 10.7 痛点七：缺少面向激光加工的专用开源方案

目前开源生态中已有成熟的机器人运动规划工具，例如 OMPL、MoveIt、Tesseract、FCL、Ruckig 等。

但它们主要面向机器人或通用运动规划，缺少面向模具激光加工的专用封装，例如：

- 激光头工具包络建模；
- 喷嘴安全距离；
- 深腔模具入口约束；
- 空移高度偏好；
- 激光加工工艺禁行区；
- CNC/激光设备轨迹输出格式。

这也是当前项目具备工程价值的原因。

## 11. 行业技术趋势

### 11.1 从直线抬高空移到智能避障空移

传统方式通常是：

```text
Z 轴抬高
  ↓
XY 快速移动
  ↓
Z 轴下降
```

这种方式简单可靠，但在深腔模具和复杂夹具场景中效率较低。

未来趋势是根据三维环境自动生成更短、更安全的空移轨迹。

---

### 11.2 从折线路径到连续光滑轨迹

未来系统不应只输出折线，而应输出曲率连续、速度连续、加速度连续的轨迹。

推荐方向：

```text
Bezier / B-spline / NURBS + jerk-limited trajectory
```

---

### 11.3 从几何避障到动力学可执行避障

未来空移规划需要同时考虑：

- 是否碰撞；
- 路径长度；
- 曲率；
- 速度；
- 加速度；
- jerk；
- 执行时间；
- 设备寿命。

也就是说，目标从：

```text
collision-free path
```

升级为：

```text
collision-free and dynamically feasible trajectory
```

---

### 11.4 从分阶段规划到联合优化

未来更先进的路线是直接优化：

```text
碰撞代价 + 路径长度代价 + 平滑代价 + clearance 代价 + 时间代价
```

可关注方法包括：

- TrajOpt；
- CHOMP；
- STOMP；
- SDF-based trajectory optimization。

---

## 12. 最具参考价值的三篇文章

## 12.1 文章一：Jerk-limited Time-optimal Speed Planning for Arbitrary Paths

### 主题

任意路径上的 jerk 限制时间最优速度规划。

### 价值

该类研究解决的是：

```text
已有几何路径如何转化为满足 jerk 限制的时间最优轨迹
```

这对激光加工设备非常关键，因为空移路径不能只满足几何避障，还必须能够被设备高速、平稳执行。

### 对当前项目的启发

- 几何路径规划后必须加入时间参数化；
- jerk 限制是高速设备轨迹规划的核心；
- 单纯最短路径不一定是最快路径；
- 轨迹执行质量需要独立评价。

### 可借鉴点

- jerk-limited speed planning；
- time-optimal path parameterization；
- 任意路径速度规划；
- 工业设备运动约束处理。

---

## 12.2 文章二：A Real-time Look-ahead Trajectory Planning Methodology

### 主题

实时前瞻轨迹规划方法。

### 价值

该研究方向与 CNC、激光切割、高速插补密切相关。其核心在于通过 look-ahead 方法提前分析后续路径，从而避免在拐角和短线段处频繁急停急启。

### 对当前项目的启发

- 空移轨迹不应逐段孤立处理；
- 需要考虑后续路径对当前速度的影响；
- B-spline / Bezier 平滑和 jerk 限制需要结合；
- 高速运动控制必须考虑实时性。

### 可借鉴点

- look-ahead 前瞻；
- G2 连续路径；
- quintic Bezier；
- jerk-limited trajectory；
- CNC 插补工程实现。

---

## 12.3 文章三：A Collision-Free, Minimized Jerk Trajectory Optimization Approach for Laser Cutting Machines

### 主题

面向激光切割机的无碰撞、最小 jerk 轨迹优化方法。

### 价值

该研究直接把两个核心目标结合起来：

```text
collision-free + minimized jerk
```

这与模具激光加工中的单段空移避障高度相关。它代表了当前从“只做避障”向“避障 + 运动质量优化”发展的趋势。

### 对当前项目的启发

- 避障和 jerk 优化应尽量联合考虑；
- 激光设备场景需要专门的轨迹质量评价；
- 平滑性直接影响设备稳定性和寿命；
- 后续可将当前项目从 OMPL + 平滑 + Ruckig 升级为联合优化框架。

### 可借鉴点

- collision-free trajectory optimization；
- minimized jerk objective；
- 激光设备专用路径质量评价；
- 避障与动力学约束结合。

## 13. 对当前项目的工程建议

结合当前项目目标，建议采用以下开发路线：

```text
阶段 1：三轴单段空移避障
  ↓
阶段 2：路径平滑与碰撞回检
  ↓
阶段 3：Ruckig jerk-limited 轨迹生成
  ↓
阶段 4：STL 模具模型与复杂夹具支持
  ↓
阶段 5：安全距离、clearance cost、路径质量评价
  ↓
阶段 6：五轴姿态约束与高级轨迹优化
```

### 13.1 第一阶段推荐技术栈

```text
几何避障：OMPL
碰撞检测：FCL
路径表达：B-spline / Bezier
轨迹生成：Ruckig
配置输入：YAML / JSON
结果输出：CSV / JSON
可视化：Python / Open3D
```

### 13.2 最小可用版本 MVP

MVP 应至少支持：

- 输入起点、终点；
- 输入 box 或 STL 障碍物；
- 输入激光头简化包络；
- 设置安全距离；
- 自动判断直线是否安全；
- 直线不安全时自动调用 OMPL；
- 输出无碰撞 raw path；
- 输出平滑 path；
- 输出 jerk-limited trajectory；
- 输出 CSV 和 JSON 报告；
- 提供可视化结果。

### 13.3 推荐优先级

优先级建议如下：

1. FCL 碰撞检测；
2. 单段直线安全性检测；
3. OMPL RRTConnect 可行路径；
4. 路径简化；
5. B-spline 平滑；
6. 平滑后碰撞回检；
7. Ruckig 轨迹生成；
8. benchmark 与可视化；
9. STL 模具复杂模型支持；
10. 五轴扩展。

## 14. 结论

模具激光加工中的单段空移避障，本质上是一个综合运动规划问题，不能只看作简单的“抬高 Z 后直线移动”。

当前行业主要技术包括：

- 几何建模与碰撞检测；
- A* / Dijkstra；
- RRT / RRT* / Informed RRT*；
- 人工势场法；
- Bezier / B-spline / NURBS 路径平滑；
- jerk-limited 轨迹规划；
- look-ahead 前瞻速度规划；
- 轨迹优化方法。

当前主要痛点集中在：

- 路径安全但效率低；
- 几何可行但设备不可执行；
- 平滑后可能碰撞；
- 五轴姿态规划复杂；
- 实时性不足；
- 安全距离难以精确保证；
- 缺少面向激光加工的专用开源工程方案。

对于当前项目，建议以：

```text
OMPL + FCL + Bezier/B-spline + Ruckig
```

作为第一阶段工程路线，先完成三轴单段空移避障 MVP，再逐步扩展到深腔模具、复杂 STL、五轴姿态和联合轨迹优化。
